EOKeySortOrdering



Inherits From: NSObject
Declared In: eoaccess/EOKeySortOrdering.h



Class Description

EOKeySortOrdering defines an order to be applied to records either during a fetch or afterwards (records can be sorted without having to re-fetch them from the data source). You usually create EOKeySortOrderings and put them into an NSArray; this array is then supplied to EOController's setSortOrdering: method. The order within the array determines the precedence when sorting. The first entry in the array specifies the primary sort order, the second entry specifies the secondary sort order, and so on.

See the setSortOrdering:, sortOrdering, and resort method descriptions in the EOController class specification for more information on using EOKeySortOrdering objects.
