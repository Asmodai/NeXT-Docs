Class Description

An OracleChannel represents an independent communication channel to the database server its OracleAdaptor is connected to. All of an OracleChannel's operations take place within the context of transactions controlled or tracked by its OracleContext. An OracleContext can manage multiple OracleChannels, and a channel is associated with only one context.

The features OracleChannel adds to EOAdaptorChannel are as follows:

Oracle-specific error handling
The ability to configure the fetch buffer
The ability to read a default list of table names from the database


Notifying the Oracle Channel's Delegate

All of the row-manipulation methods notify the adaptor channel's delegate before and after performing their operations. The OracleChannel class provides a delegate method for handling errors: oracleChannel:willReportDatabaseError:. This delegate message is invoked whenever the channel encounters an error reported by the Oracle server. The delegate uses the methods cursorDataArea, hostDataArea, and logonDataArea to retrieve Oracle-specific data structures from the channel and context to determine what error has occurred. The hostDataArea and logonDataArea methods are declared in the OracleContext class.

Class Methods

oracleTableNamesSQL
+ (NSString *)oracleTableNamesSQL

Returns the SQL statement that will be executed when building a default model.



setOracleTableNamesSQL:
+ (void)setOracleTableNamesSQL:(NSString *)sql

Sets to sql the SQL statement that will be used to return a list of table names from the database. By default, this list is the result of the SQL statement:

SELECT TABLE_NAME FROM USER_TABLES ORDER BY TABLE_NAME

This setting is used by all OracleChannels in an application. You can specify a different SQL statement using the dwrite command, for example:

% dwrite Oracle7Adaptor OracleTableNamesSQL "SELECT TABLE_NAME FROM..."

One you use setOracleTableNamesSQL: to specify a setting, it supersedes values set with the dwrite command.



Instance Methods

cursorDataArea
(Cda_Def *)cursorDataArea

If the channel is connected, returns an Oracle-specific data structure describing characteristics of the channel. Otherwise, returns NULL. This method is commonly used with the delegate method oracleChannel:willReportDatabaseError: to determine why an error occurred.



fetchBufferLength
(unsigned int)fetchBufferLength

Returns the size, in bytes, of the fetch buffer. The larger the buffer, the more rows can be returned for each round trip to the server.



setFetchBufferLength:
(void)setFetchBufferLength:(unsigned int)anInt

Sets the size, in bytes, of the fetch buffer. The larger the buffer, the more rows can be returned for each round trip to the server.



Methods Implemented by the Delegate

oracleChannel:willReportDatabaseError:
(BOOL)oracleChannel:(OracleChannel *)channel
willReportDatabaseError:(NSString *)error

This delegate message is invoked whenever the channel encounters an error reported by the Oracle server. The delegate uses cursorDataArea, hostDataArea, and logonDataArea to retrieve Oracle-specific data structures from the channel and context to determine what error has occurred. (The hostDataArea and logonDataArea methods are declared in the OracleContext class.) error is the text of the error message that will be sent to the adaptor's reportError: method. The delegate can return NO to prevent the channel from calling reportError:. See your Oracle client library documentation for more information about the data structures returned by the cursorDataArea, hostDataArea, and logonDataArea methods.
