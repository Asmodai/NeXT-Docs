NXFormatterViewEditing



Adopted By: no Enterprise Objects Framework classes
Declared In: eointerface/NXEditableFormatter.h



Protocol Description

This protocol declares the method that an editable NXFormatter (such as NXEditableFormatter) sends to the view it's drawing in when it finishes editing a value. If you implement a view class that uses NXFormatters, it should conform to this protocol.



Instance Methods

formatterDidEndEditing:endChar:
(BOOL)formatterDidEndEditing:sender endChar:(unsigned short)whyEnd

Invoked by an editable formatter it finished editing, typically when the user presses one of the keys that moves to another field (Return, Tab, or Shift-Tab). The field that was being edited is redrawn, and the field below, after, or before it is selected, depending on the value of whyEnd, which may be NX_RETURN, NX_TAB, or NX_BACKTAB. Normally returns YES; returns NO if invoked while no field is being edited.
