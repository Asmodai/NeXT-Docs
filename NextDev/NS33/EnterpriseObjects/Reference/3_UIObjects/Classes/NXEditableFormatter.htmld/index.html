NXEditableFormatter



Inherits From: NXFormatter : Object
Declared In: eointerface/NXEditableFormatter.h



Class Description

An NXTableView uses NXEditableFormatter object for the display and editing of textual data in an NXTableView. For faster, read-only display of text, use NXTextFormatter. To display images in an NXTableView, use NXImageFormatter.

editFieldAt::inside:inView:withAttributes::usePositions::onEvent:
(BOOL)editFieldAt:(unsigned int)rowIndex
:(unsigned int)columnIndex
inside:(NXRect *)frameRect
inView:aView
withAttributes:(id <NXTableVectors>)rowAttributes
:(id <NXTableVectors>)columnAttributes
usePositions:(BOOL)rowPosFlag
:(BOOL)columnPosFlag
onEvent:(NXEvent *)theEvent

Begins an editing session for the field of aView identified by rowIndex and columnIndex. You never invoke this method directly; it's invoked automatically by the NXTableView when a field needs to be edited. When editing ends this method sends formatterDidEndEditing:endChar: to aView (this method is described in the NXFormatterEditing protocol specification).

Returns YES if the field is successfully edited, NO otherwise (for example, if aView can't be made the first responder).

Note:  aView isn't an NXTableView, but a private view used by the NXTableView.

See also:  endEditing, abortEditing, drawFieldAt::inside:inView:withAttributes::usePositions::



endEditing
(BOOL)endEditing

Finishes editing in the field indicated by an editFieldAt::... message, typically when the user presses Return or Tab or clicks in another field (thereby indicating that editing in the receiver is complete). Returns YES if successful, NO otherwise.
