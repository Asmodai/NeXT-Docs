EODatabaseCustomValues
(informal protocol)


Category Of: NSObject
Declared In: eoaccess/EOCustomValues.h



Category Description

The Enterprise Objects Framework uses the methods of the EOCustomValues and EODatabaseCustomValues categories to package text or binary data into custom value objects. A custom value class can be specified for a database column read through the access layer, or for a user interface object accessed through the interface layer. For example, you can map a binary column in a database to an NXImage subclass, or a text entry field in your application to a Currency class.

EOCustomValues declares the methods for custom value classes based on text data; EODatabaseCustomValues declares the methods for custom value classes based on binary data. Your custom value class can implement either or both of these categories' methods, depending on how its values will be accessed by the various components of the Framework and your application.



Implementing EODatabaseCustomValues

Your custom value objects are initialized with initWithData:type:, which offers the binary data and a string identifying the format of that data; for example, "TIFF". The value object can use this type string to determine how to interpret the data. When your object's data is needed in its raw form, it's sent a dataForType: message specifying the type needed. Your object must produce an NSData object containing bytes in the named format.

In addition to implementing the methods declared in this category, your custom value class must meaningfully implement the description method if its value is to be displayed in a text control. description is the standard method for getting a reasonable text representation of an object, described in the NSObject class specification of the Foundation Kit Reference. Your custom class must also implement this method if it's to be used for an enterprise object's primary key.

Instance Methods

dataForType:
(NSData *)dataForType:(NSString *)typeString

Returns the value of the receiver as a data object in the format indicated by typeString. This method is used to convert a custom type to an NSData object for writing to the database. For example, this code excerpt produces an image in Tagged Image File Format:

NSData *imageData = [myCustomValue stringForType:@"TIFF"];

See also:  valueType (EOAttribute class)



initWithData:type:
initWithData:(NSData *)data type:(NSString *)typeString

Initializes the receiver to contain the value in data. typeString, if non-nil, indicates the format of the contents of data; for example, "TIFF". Returns self.

This method is used to create a custom type from an NSData object read from the database.
