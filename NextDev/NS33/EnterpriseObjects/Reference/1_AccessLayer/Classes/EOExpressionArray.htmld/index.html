EOExpressionArray



Inherits From: NSMutableArray : NSArray : NSObject
Conforms To: NSCopying
Declared In: eoaccess/EOExpressionArray.h



Class Description

EOExpressionArray is a subclass of NSMutableArray that composes an expression value from the expression values of its elements when you send it an expressionValueForContext: message (see the EOExpression informal protocol specification). An expression value is a representation of an object suitable for inclusion in a SQL statement. This class is used internally by the Framework; you should never need to create or use one yourself.

In addition to its array elements, an EOExpressionArray keeps a trio of strings that it uses to format its expression value in certain cases. It has a prefix that precedes the elements' expression values, an infix that gets inserted between each pair of expression values, and a suffix that's appended to the elements' expression values.

If you provide a context when you ask for an EOExpressionArray's expression value and its first element is an EORelationship, its elements are assumed to be a data path ending in an attribute, and the expression value depends on how the context handles that data path. If you don't provide a context, however, the EOExpressionArray builds a string with the expression values of its elements and its prefix, infix, and suffix.

Suppose an expression array contains the following items:

array elements: EOAttributes named "lastName", "firstName", and "address"
prefix: "("
infix: ", "
suffix: ")"

The expression value of this EOExpressionArray is "(lastName, firstName, address)".

Instance Methods

expressionValueForContext:
(NSString *)expressionValueForContext:(id <EOExpressionContext>)context

If context is non-nil and the receiver's first element is an EORelationship, sends the context an expressionValueForAttributePath: message with self as the argument and returns the resulting expression value; this allows the elements to be property aliased by the context. Otherwise, returns a string with the prefix and suffix on either end, and with the expression values of the receiver's elements between, each separated from the others by the infix string. See the class description for an example.

Returns nil if the receiver is empty.

referencesObject:

Returns YES if the receiver contains anEO or if any EOExpressionArray contained by the receiver contains anEO, NO otherwise. Pointer equality, not isEqual:, is used to compare objects.

This method is used by the modeling objects for consistency checking. You should never call this method yourself.
