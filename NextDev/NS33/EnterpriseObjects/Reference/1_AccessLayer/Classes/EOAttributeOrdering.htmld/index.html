EOAttributeOrdering



Inherits From: NSObject
Conforms To: NSCopying
Declared In: eoaccess/EOAttributeOrdering.h



Class Description

EOAttributeOrdering defines the order that an EOAdaptorChannel or EODatabaseChannel fetches rows or objects in. You usually create EOAttributeOrderings and put them into an NSArray for a select operation. The order within the array determines the precedence when sorting. For example, the following code excerpt selects Employee objects so that they'll be fetched alphabetically by last name and then first name:

EOQualifier *empQualifier;         /* Assume this exists. */
EOAttribute *firstNameAttribute;   /* Assume this exists. */
EOAttribute *lastNameAttribute;    /* Assume this exists. */
EODatabaseChannel *myDBChannel;    /* Assume this exists. */
EOAttributeOrdering *firstNameOrdering;
EOAttributeOrdering *lastNameOrdering;
NSArray *nameOrder;
BOOL ok;

firstNameOrdering = [EOAttributeOrdering
attributeOrderingWithAttribute:firstNameAttribute
ordering:EOAscendingOrder];
lastNameOrdering = [EOAttributeOrdering
attributeOrderingWithAttribute:lastNameAttribute
ordering:EOAscendingOrder];

nameOrder = [NSArray arrayWithObjects:lastNameOrdering,
firstNameOrdering, nil];

ok = [myDBChannel selectObjectsDescribedByQualifier:empQualifier
fetchOrder:nameOrder];

if (ok) {
/* Go on to fetch objects. */
}
