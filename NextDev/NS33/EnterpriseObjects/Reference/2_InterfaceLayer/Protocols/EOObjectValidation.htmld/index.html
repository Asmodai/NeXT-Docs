EOObjectValidation



Adopted By: no Enterprise Objects Framework classes
Declared In: eointerface/EOController.h



Protocol Description

The EOObjectValidation protocol declares a method for notifying an enterprise object that it's about to be sent to its persistent storage. An enterprise object class can adopt this protocol to allow additional validation of values sent from user interface objects.



Instance Methods

prepareForDataSource
(BOOL)prepareForDataSource

Sent by an EOController from saveToDataSource to inform the receiver that it's about to be written to the controller's data source. The receiver can take whatever action it needs and return YES to allow itself to be saved or NO to disallow it (possibly causing saveToDataSource to abort).

See also:  controller:object:failedToPrepareForDataSource: (EOController class, "Methods Implemented by the Delegate")
