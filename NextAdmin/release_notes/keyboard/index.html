3.3 Release Notes:
The Keyboard Application




This file contains release notes for the 3.3 release of the Keyboard application (located in /NextDeveloper/Demos).




New Features

The following new features have been added to Keyboard since Release 3.2:

Keyboard now supports the assignment of sequences of keys to individual keys.
Keyboard is now localized.





Bugs Fixed in Release 3.3

These bugs have been fixed:

You can now drag and drop characters directly onto the large keycap in addition to dragging onto the keyboard.
Double-clicking on keymapping files would sometimes cause a blank keymap to come up and then Keyboard would spin or crash.
Revert to Saved left the window in an inconsistant state if modifer keys were enabled.
You couldn't make a key into a dead key.
Keyboard always started its open panel in ~/ rather than ~/Library/Keyboards.  It now uses the current document as the directory to start from and ~/Library/Keyboards if no file is open.  The Save panel also starts in a more rational location.
The key inspector area of a document would flash horribly when you dragged a character over a keyboard.
Revert to Saved on a new document crashed Keyboard.
